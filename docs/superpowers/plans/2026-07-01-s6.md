# Plan — s6: commit a Node-builtin test harness for the client JS

## Goal (this slice only)
Bring the inline JS in `scripts/dashboard_assets/index.html` under a **committed**
test harness using **Node built-ins only** (`node:test` + `node:assert`; coverage via
`node --test --experimental-test-coverage`). Make the pure/DOM-producing functions
testable **without** breaking the self-contained single-file page (no CDN, no npm, no
build step; the page must still work opened directly in a browser). Add the committed
test file and a Node test step to `.github/workflows/validate.yml`.

## Design: how the single-file page becomes testable without breaking

The inline `<script>` currently (a) defines ~32 functions, then (b) grabs DOM refs at
top level (`document.getElementById(...)`, lines 156–159), (c) initializes app state at
top level (`let view = parseHash();`, **line 163** — this dereferences `location.hash`),
and (d) bootstraps the app at load (`window.addEventListener("hashchange", ...)`,
`setInterval(...)`, `enterView()`, lines 506–517). Loading that whole script under Node
would crash on `document`/`location`/`window`.

### TRANSPORT DECISION (folds the Architect's OBJECT — the only blocker)
The Architect empirically proved that loading the extracted script via `node:vm`
produces **zero coverage attribution** for the client code (V8 does not instrument
`node:vm`-evaluated strings), so `--experimental-test-coverage` would be decorative.
I re-verified this and confirmed the fix: code loaded as a **real on-disk module that
Node `import()`s** IS instrumented and attributed. Therefore the harness does NOT use
`node:vm`. Instead it:
1. reads `index.html`, extracts the single `<script>…</script>` body,
2. strips the browser-inert UMD export tail and appends a named ESM `export { … }`,
3. writes the result to a temp `.mjs` under `os.tmpdir()` (via `node:fs`/`node:os`),
4. `await import(pathToFileURL(temp).href)` — a REAL module, so V8 attributes coverage
   to it (verified: the extracted module shows in the coverage table at 100%).

This keeps the slice to exactly its three declared files (no committed sibling module
the browser would have to fetch), keeps the page truly self-contained (the browser
never loads anything but the inline script), AND makes `--experimental-test-coverage`
real. The temp module's body IS the page's inline JS verbatim (minus the export shim),
so coverage is genuinely coverage of the client JS.

**Approach — a minimal dual-mode boundary inside the SAME inline script (zero new
files loaded by the page, no bundler, no network):**

1. **Guard EVERY browser-only top-level side effect** behind a single
   `const HAS_DOM = typeof document !== "undefined" && typeof window !== "undefined";`.
   The guard MUST cover all three top-level touch points (Architect/Guardian/Pragmatist
   all flagged that line 163 was omitted from the first draft):
   - DOM-ref capture, lines 156–159 (`$view/$crumbs/$freshness/$freshText`);
   - **`let view = parseHash();` at line 163** — give `view` a Node-safe default
     (`let view = HAS_DOM ? parseHash() : { kind: "overview" };`) so it never touches
     `location` under Node;
   - the bootstrap block, lines 506–517 (`window.addEventListener`, `setInterval`,
     `enterView()`), wrapped in `if (HAS_DOM) { … }`.
   In a browser `HAS_DOM` is true → identical behavior. Under Node it is false → no
   bootstrap, nothing touches `document`/`location`/`window`. No seam is ever assigned
   to `window`/`globalThis` (Guardian: no stray globals).
   The `$` refs stay `null` under Node; tests call the *builder* functions directly and
   never the `$view`-mutating orchestrators. `renderOverview`/`renderDetail`/`poll`/
   `applyResponse`/`enterView`/freshness/fetch are **OUT of scope for s6** (Skeptic:
   state this explicitly) — the poll/fetch/freshness state machine is not tested here.

2. **Expose the testable surface.** At the very end of the script add a guarded UMD
   export tail that is inert in a real browser (`module` is undefined → no-op) and in
   real Node ESM; the harness strips it and substitutes an ESM export during extraction:
   ```js
   /* test-export */ if (typeof module !== "undefined" && module.exports) {
     module.exports = { labelClass, sha7, truncate, groupRunsByRoot, parseHashFrom,
       el, overviewCard, rootGroupSection, sliceRow, __setDocument };
   }
   ```
   Export list TRIMMED to exactly the tested surface (Pragmatist): drop `pill`,
   `countPills`, `wavesSection`, `sliceTableSection` unless a test asserts them. A
   `/* test-export */` marker gives the extractor a stable strip/replace anchor.

3. **DOM seam for `el()`.** `el()` calls `document.createElement`. Introduce a
   module-scoped `_doc` defaulting to the global `document` (browser) and swappable via
   an exported `__setDocument(doc)` for tests; `el()` uses `_doc.createElement`. The
   test supplies a small pure-JS DOM shim implementing exactly the API in use. The shim
   MUST implement variadic `append(...children)` accepting BOTH element nodes and text
   nodes, and text nodes need a serialization path so textContent assertions work
   (Architect: `node.append(a,b,c,d)` and `$crumbs.append(el, createTextNode(...))` are
   used at lines 226, 280, 290, 357). Methods needed: `createElement`, `createTextNode`,
   node `.className`/`.textContent`/`.hidden`/`.tagName`/`.children`/`.appendChild`/
   `.append(...)`/`.addEventListener`. Behavior-preserving in the browser (`_doc ===
   document`).

4. **`parseHash()` seam.** Extract the pure core `parseHashFrom(hashString)`
   (string→view object) and have `parseHash()` call `parseHashFrom(location.hash)`.
   Browser behavior identical; tests exercise `parseHashFrom` for the namespaced-id
   round-trip.

**Net page impact:** all changes are internal seams + guards; opening `index.html`
directly in a browser behaves exactly as before (no fetch, no module import, no build).
The read-only / no-new-fetch-verb boundary is untouched — the single existing `fetch`
GET (line 439) stays exactly as-is; the harness adds no network verb.

## Test harness (`scripts/dashboard_assets/index.test.mjs`, committed)
- Node built-ins only: `import test from "node:test"; import assert from "node:assert/strict";`
  plus `node:fs`, `node:os`, `node:path`, `node:url`. NO `node:vm`, NO third-party dep.
- Header docstring mirrors `test_dashboard_server.py`'s intent (Historian): "sole
  automated guard on the client's behavior; Node built-ins only."
- Loader (real-module transport, per TRANSPORT DECISION): read `index.html`, extract the
  single `<script>` body, strip the `/* test-export */` UMD tail, append
  `export { labelClass, sha7, truncate, groupRunsByRoot, parseHashFrom, el, overviewCard,
  rootGroupSection, sliceRow, __setDocument };`, write to a temp `.mjs`, `await import()`
  it. Install the DOM shim via `__setDocument`. No `document`/`location`/`window` exist
  in the Node process → the `HAS_DOM` guard keeps the bootstrap dormant (Guardian: also
  assert no network activity / bootstrap stays dormant on load).

## Coverage gate note (Skeptic decision — recorded, deferred numeric floor to s7)
`--experimental-test-coverage` runs and REPORTS coverage on CI Node 20, but the
per-metric threshold flags (`--test-coverage-lines/functions/branches`) arrived in
Node 22 and are unavailable on Node 20 — so there is NO numeric floor gate in s6.
**s6 success = the named in-scope functions covered by passing behavioral tests, and
the CI step fails on any test failure or zero discovered tests.** The numeric JS
coverage floor is a deliberate handoff to s7, which should adopt the SAME durable-gate
philosophy already established for Python in `measure_coverage.py` (explicit measured
surface, per-target integer floor, fail-closed on empty discovery — the `MIN_TESTS`
lesson). Record this decision + handoff in `decisions-log.md`.
