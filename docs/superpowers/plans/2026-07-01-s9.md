## TDD tasks (each: add test(s), run, confirm the targeted line is now hit)

Follows the existing file's conventions exactly (stdlib unittest, `tempfile`, `slice_obj`/
`write_dag` builders, `make_handler` in-thread mirror, ephemeral port 0 real sockets). Each
new test asserts REAL observable behavior ‚Äî never an assert-nothing line-hit.

1. **`resolve_within` ValueError branch (L95/L97).** Pure-function unit test in
   `ContainmentTests`. IMPORTANT (architect finding): a crafted `relpath` cannot trigger this on
   macOS/Linux ‚Äî inside `resolve_within`, both `real_root` and `candidate` are `os.path.realpath`'d
   (absolute) BEFORE `commonpath` is called, so the mixed-abs/relative case cannot arise and
   cross-drive does not exist on POSIX. The deterministically-reachable mechanism is to
   `unittest.mock.patch("dashboard_server.os.path.commonpath", side_effect=ValueError)` for the
   duration of one call and assert `ds.resolve_within(root, "ok.txt") is None` ‚Äî i.e. the
   deny-on-error SECURITY contract (any containment-check failure denies). Stdlib-only; this
   forces the deny branch, it does NOT stub/weaken a security primitive (the real realpath +
   isabs + null-byte checks still run; only the final commonpath comparison is made to error).

2. **Malformed-dag `slices` not a list (L168).** In `ScanRunsTests` (or a fresh tmp): write a
   `dag.json` whose `slices` is an object/number (valid JSON, wrong type). Assert the run is
   emitted as `{"status": "unreadable"}` and siblings survive ‚Äî the same graceful degrade the
   corrupt-JSON test already guards, via a different raise path.

3. **Unsatisfiable-deps wave `break` (L231).** Build a dag with a dependency cycle (or a dep on
   a non-existent id). Assert `_derive_waves` (via `scan_runs`) terminates and the unsatisfiable
   pending slices do NOT appear in any wave (no infinite loop, honest omission).

4. **Escalation-parser branches (L301, L325, L337, L346).** Add focused unit tests calling the
   pure helpers directly and/or via `scan_runs`:
   - a `## [`-prefixed line with NO closing `]` -> `_parse_escalation_header` returns
     `(None, "", "")` and `_iter_escalation_headers` `continue`s (block ignored, no crash).
   - a header with a bracket token but NO `(status: ...)` and NO filled Answer -> state `""`
     (`_header_state` return "") -> the slice is neither awaiting-human nor redispatch-pending.
   - `_has_filled_answer([...])` with body lines that contain no `Answer:` -> False.

5. **`_owning_root` / `_resolve_run_dir` miss branches (L467, L483-487).** Direct in-thread
   calls on a MULTI-root bound handler (build via `build_server([root_a, root_b], ...)`):
   - a namespaced id with a bogus/unknown root key -> `_owning_root` returns `("", None, None)`
     and `_resolve_run_dir` returns `("", None)` -> detail request 404 (uniform miss).
   - a valid key with a missing bare id -> matched-key branch, `(root_key, None)`.
   Assert via the existing no-oracle 404 contract where a socket path is clearer.

6. **Handler entry points `do_GET`/`do_HEAD`/`_reject_method` (L397/400/415).** In-thread mirror
   in `HandlerUnitTests`: give the handler a `BytesIO` wfile + real allowlisted Host, set
   `self.path`, call `do_GET()` / `do_HEAD()` directly and assert the emitted wire bytes
   (status line, body present for GET / absent for HEAD). Call one rejected verb entry
   (`do_POST`, which aliases `_reject_method`) and assert `405 method not allowed` on the wire.
   These run the REAL `_route`/`_emit`/`_host_allowed`; nothing security-relevant is stubbed.
