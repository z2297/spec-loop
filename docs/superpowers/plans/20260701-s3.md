## Uncovered paths to cover (behavior, not implementation)

The trace shows the missed executable lines are the error/branch arms plus `main()`.
Group into TDD-sized tasks:

### Task 1 — Minimal `main(argv=None)` refactor (endorsed)
- Change signature to `main(argv=None)`. Fold: `argv = sys.argv[1:] if argv is None
  else argv`, then read root as `argv[0] if argv else "."` — do NOT reintroduce a raw
  `sys.argv[1]` index [guardian]. Keep the not-a-directory message, the `OK:`/`FAILED:`
  strings, and both exit codes byte-identical. `__main__` guard stays `sys.exit(main())`.
  The refactor touches ONLY `main()` — no `validate_*` method changes [guardian/architect].
- Mirrors `pr_resolver.py:459` / `dashboard_launcher.py:536` precedent [historian].
- This is the ONLY source edit. It unlocks the three `main()` exit-code paths for tests
  without process spawning or `sys.argv` mutation.

### Task 2 — `main()` exit-code + output tests (new `MainCliTest`)
Capture stdout/stderr via `contextlib.redirect_stdout/redirect_stderr(io.StringIO())`
verbatim, per the repo idiom in `test_dashboard_launcher.py` [historian].
- not-a-directory arg (a path that isn't a dir) → returns 1, prints `error: not a
  directory` to stderr.
- valid repo (reuse `write_plugin`, pass its tmp root via `main([str(root)])`) →
  returns 0, prints `OK:` to stdout.
- invalid repo (missing marketplace.json, tmp root via `main([str(root)])`) → returns 1,
  prints `FAILED:` + the error list to stderr.
- default arg: `chdir` into a fresh `TemporaryDirectory` (restore cwd via
  `addCleanup`), call `main([])`, assert it returns an `int` without raising — do NOT
  run against the live checkout and do NOT assert a fixed exit code [guardian/historian INVARIANT].

### Task 3 — marketplace-level structural failures (new `MarketplaceStructureTest`)
Drive via `vm.Validator(root).run()` on tmp fixtures; assert `ok is False` and the
naming/substring of the error:
- missing `.claude-plugin/marketplace.json` → "missing required file".
- invalid JSON in marketplace.json → "invalid JSON".
- non-kebab / non-string `name` → "'name' must be a kebab-case string".
- missing/empty `owner.name` → "'owner.name' is required".
- `plugins` not a list / empty → "'plugins' must be a non-empty array".
- `metadata.pluginRoot` honored: source is a BARE name (`"source": "demo"`, NOT
  `"./demo"`) so line 113 (pluginRoot prefixing) is actually reached, plugin dir laid
  down under the pluginRoot, entry passes [architect].

### Task 4 — plugin-entry failures (new `PluginEntryTest`)
- entry not an object → "must be an object".
- non-kebab entry name.
- duplicate plugin name → "duplicate plugin name".
- missing string `source` → "'source' (relative path) is required".
- `source` containing `..` → "path traversal".
- source path not a directory → "does not resolve to a directory".
- missing `plugin.json` → "missing …/plugin.json".

### Task 5 — object-source validation (new `SourceObjectTest`)
- unknown `source.source` kind → "must be one of".
- each kind missing a required key (e.g. github without repo) → "requires string".
- empty `ref` → "'source.ref' must be a non-empty string".
- bad `sha` (not 40 hex) → "must be a 40-char commit SHA".
- a fully valid object source (github w/ repo, good ref+sha) → passes.

### Task 6 — plugin manifest + frontmatter failures (new `PluginManifestTest`)
- manifest name non-kebab.
- manifest name != marketplace entry name → "does not match marketplace entry".
- version mismatch entry vs manifest → "version mismatch".
- MALFORMED `plugin.json` (invalid JSON bytes) → covers line 155 (`manifest is None`
  early return) AND `load_json`'s invalid-JSON arm on the manifest path [architect].
- read-only command whose frontmatter OMITS `allowed-tools` entirely → PASSES; covers
  `_allowed_tools` line 226 (`not isinstance(raw, str)` → empty set) [architect].
- SKILL.md frontmatter name != its directory → "does not match its directory".
- missing required frontmatter key (agent w/o name) → "missing required".
- missing frontmatter block (no `---`) → "missing YAML frontmatter" (plant via
  `write_frontmatter_plugin` so `validate_frontmatter_files` is actually reached).
- unterminated frontmatter block → "unterminated frontmatter block".
- `_rel` ValueError fallback (line 47-49): covered incidentally — no bespoke
  private-helper test [pragmatist]. If it stays dark after the above, add ONE test
  that routes a foreign path through an observable error (e.g. a source resolving
  outside root) rather than calling `_rel` directly.
