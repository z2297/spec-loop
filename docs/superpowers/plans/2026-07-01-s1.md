# Slice s1 — coverage foundation

## Goal
Deliver the measurement + enforcement foundation: run the Python unittest suite in
CI, add a stdlib-`trace`-only coverage tool with an auditable OMIT manifest, wire a
deterministic no-regression gate into CI at the CURRENT measured floor, and cover
the tool's own pure logic with unittest.

## Constraints (folded from intake council + human resolution)
- STDLIB ONLY: stdlib `trace`; NO coverage.py / pytest / third-party.
- Deterministic gate (fixed floors, no per-run flake); python 3.12; no Docker; no network.
- Match existing conventions (stdlib `unittest`).
- Repo-scoped enforcement in `validate.yml`; never touch machine-global quality-gate.json.
- Floors set from a REAL measurement run in the worktree.

## Files (stay within these)
- `.github/workflows/validate.yml`
- `scripts/measure_coverage.py`
- `scripts/coverage_omit.txt`
- `scripts/test_measure_coverage.py`

## Design

### Executable-line detection (principled, not regex)
Compile each target file with `compile(src, path, "exec")`, then recursively walk
`code.co_consts` collecting every nested code object. Union the line numbers from
each code object's `co_lines()` (the 3-tuple `(start, end, lineno)` iterator, py3.10+),
dropping any lineno that `is None` **OR `< 1`** — `co_lines()` emits a synthetic
`lineno == 0` for the module code object's prologue even for real on-disk files
(confirmed on py3.14 by architect + guardian). Line 0 is not a real source line and
is version-dependent, so it must never enter the executable set. This yields the set
of lines the interpreter can actually land on — bytecode-derived truth, not a heuristic.

### Executed-line detection
Run the unittest suite in-process under `trace.Trace(count=1, trace=0)` via
`tracer.runfunc(...)`. Read `results.counts` — a dict keyed `(abs_filename, lineno)`
with hit counts. `trace` reports ABSOLUTE paths; normalize each to the repo-relative
`scripts/<name>.py` key form used by floors and OMIT at the single seam where the
executed set is built (unit-tested, so a path-form drift fails loudly instead of
silently reporting 0 coverage). Executed set for a file = normalized-key linenos with
count > 0, with the same `lineno >= 1` filter applied.

### Suite runner (must pass before coverage is trusted)
The in-process runner loads tests via `unittest.defaultTestLoader.discover("scripts",
"test_*.py")` (same set the CI test step runs) with `scripts/` inserted on `sys.path`
so both target modules and their `test_*.py` resolve by basename (matching existing
import convention). It runs under `tracer.runfunc(runner.run, suite)` and asserts
`result.wasSuccessful()`; if the suite fails, the tool prints the failure and
`sys.exit(1)` WITHOUT reporting coverage — coverage of a red suite is meaningless.

### OMIT manifest (`scripts/coverage_omit.txt`)
Line-oriented text file. Format per non-blank, non-`#` line:
`relpath:START[-END]  # rationale`
Blank lines and full-line `#` comments ignored. **Every data entry MUST carry a
`# rationale`; `parse_omit` raises on a malformed entry or one missing a rationale**
(guardian anti-escape-hatch: exclusions stay auditable, cannot be silently widened to
lift the ratio). Only legitimate exclusions: `__main__` shims and the blocking
`serve_forever()`/daemon tails. Excluded lines are removed from BOTH the executable set
and the executed set before ratio computation, so they neither help nor hurt.

### Report + gate
For each `scripts/*.py` (excluding the test files themselves and measure_coverage.py's
own scope decision — see below), compute executed/executable after OMIT. Print a
per-file table + total. Compare per-file to per-file floors and total to total floor
(floors defined as module-level constants in measure_coverage.py, keyed by relpath).
`sys.exit(1)` if any floor is breached; else `sys.exit(0)`.

Target scope (floors keyed by full relpath): `scripts/dashboard_launcher.py`,
`scripts/dashboard_server.py`, `scripts/pr_resolver.py`, `scripts/release.py`,
`scripts/validate_marketplace.py`. Exclude `test_*.py` and `measure_coverage.py` from
the MEASURED set (the tool measures product code; its own coverage is asserted by
test_measure_coverage.py — documented contract, confirmed accepted by skeptic). Document
this in the file.

### Determinism across interpreters (guardian, load-bearing)
Local is py3.14; CI enforces on py3.12. Floors are captured from a REAL worktree
measurement, then **rounded DOWN to a safe integer margin** so a minor cross-version
executable-set drift cannot make a floor unreachable on 3.12 (false red → CI wedge).
The CI coverage step pins `python-version: "3.12"` (already the workflow default) and
the gate is deterministic (fixed integer floors, no time/random lines in scope). The
gate must PASS on the same commit that sets the floors (skeptic self-consistency) —
verified in step 7.

## TDD steps (bite-sized)
1. Create OMIT manifest with the two known categories (main shims + serve_forever tail),
   one rationale per line.
2. Write `test_measure_coverage.py` first (header convention per test_validate_marketplace.py:
   docstring + `sys.path.insert(0, str(Path(__file__).resolve().parent))` +
   `import measure_coverage as mc  # noqa: E402` + `unittest.main()` tail) for the pure
   helpers against synthetic inputs:
   - `executable_lines(source)` returns bytecode-landable lines for a small snippet AND
     never contains 0 (asserts the lineno<1 filter).
   - `normalize_key(abs_path)` maps an absolute path to `scripts/<name>.py`.
   - `parse_omit(text)` returns the correct {relpath: set(lines)}, ignores comments/blanks,
     and RAISES on a malformed / rationale-less entry.
   - `apply_omit(...)` subtracts omitted lines from both sets.
   - `evaluate(per_file_stats, floors, total_floor)` returns pass/fail + which floors breached.
3. Implement `measure_coverage.py` pure helpers to green those tests (Red→Green→Refactor).
4. Implement the trace-driven suite runner + path normalization + report + `main()`/exit
   logic: discover via `defaultTestLoader.discover("scripts","test_*.py")`, assert
   `wasSuccessful()` (exit 1 + no coverage report if the suite fails), normalize counts keys.
5. RUN it in the worktree to obtain REAL per-file + total numbers; reconcile against the
   recorded baseline (decisions-log: 72.4% total; release.py 0.0%, dashboard_server 67.0%,
   validate_marketplace 69.2%, dashboard_launcher 92.9%, pr_resolver 97.5%); set floors
   rounded DOWN to a safe integer margin (deterministic, py3.12-safe).
6. Add CI steps to `validate.yml` (python-version already pinned 3.12): (a) run unittest
   suite via exact `python3 -m unittest discover -s scripts -p 'test_*.py'`, (b) run
   `python3 scripts/measure_coverage.py` gate. Insert AFTER setup-python and the marketplace
   validation, WITHOUT disturbing existing step ordering.
7. Verify (self-consistency): run full unittest suite + measure_coverage locally on the
   same commit that sets the floors; confirm the gate PASSES green.

## Risks / mitigations
- py3.14 local vs py3.12 CI: `co_lines()` exists in both; executable-line sets are stable
  across these versions for this code. Set floors with a safety margin (round down).
- Determinism: floors are fixed integers; no random/time-dependent lines in scope.
- Not blocking on unwritten tests: floors set to CURRENT measured baseline, not 100%.
