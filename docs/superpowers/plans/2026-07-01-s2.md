# Plan: s2 — test scripts/release.py (0.0% → well above floor)

## Goal
Add `scripts/test_release.py` covering `scripts/release.py`'s pure file-transform
logic, asserting resulting file CONTENT (behavior), not line-hits. All tests run
against an isolated tmp root seeded with fixture manifests. Optionally apply the
single endorsed `main(argv=None)` refactor to `release.py` so argparse paths are
testable without `sys.argv` hacking.

## Constraints (from slice)
- Isolated tmp root per test (`TemporaryDirectory`); NEVER mutate the real repo.
- STDLIB ONLY (unittest); match `scripts/test_*.py` conventions.
- `release.py` is PURE — no subprocess/git — so no mocking.
- Do NOT edit `measure_coverage.py` / `coverage_omit.txt` / `validate.yml`.
- Merge into `alpha`.

## Steps (TDD, bite-sized)

### Step 0 (refactor precondition): `main(argv=None)` in release.py
`main()` currently calls `ap.parse_args()` (reads `sys.argv`). To test the
argument/validation paths (`--notes`, invalid semver, missing marketplace,
stable+prerelease error, beta-without-suffix warning) without mutating global
`sys.argv`, change the signature to `def main(argv=None):` and call
`ap.parse_args(argv)`. The `__main__` shim stays `sys.exit(main())` (argv=None →
argparse reads sys.argv, unchanged behavior). This is the only endorsed refactor;
it is behavior-preserving.

### Step 1: fixture builder + test scaffold
Write `seed_repo(root)` helper that lays down under a tmp root:
- `plugins/spec-loop/.claude-plugin/plugin.json` with a known `version`
- `.claude-plugin/marketplace.json` with channel entries + an existing archive entry
- `CHANGELOG.md` with `[Unreleased]` section (with body) + link refs
Match the real manifest shapes closely enough that the transforms exercise
their real branches.

### Step 2: `bump_plugin_version` — content assertions
- bumps version when different (returns True; plugin.json version == new)
- idempotent no-op when already at version (returns False; unchanged)

### Step 3: `add_archive_entry` — content assertions
- inserts pinned `spec-loop-<kebab>` entry with `ref == v<version>`, source
  git-subdir/url/path correct, description mentions `v<version>` (returns True)
- inserts AFTER channel entries, AHEAD of existing archive entries (ordering)
- idempotent no-op when entry already present (returns False)

### Step 4: `roll_changelog` — content assertions
- rolls `[Unreleased]` → dated `[<version>]`, adds fresh empty `[Unreleased]`,
  updates link refs (compare/tag URLs) (returns True)
- idempotent no-op when `[<version>]` section already present (returns False)
- WARNING no-op when no `[Unreleased]` section (returns False)

### Step 5: `extract_notes`
- returns the stripped body of the version section (excludes heading, stops at
  next `## [`)
- returns "" for a missing version

### Step 6: `cut_release`
- stable channel: bumps + adds archive entry + rolls changelog (all three effects)
- beta/alpha channel: bumps only (no archive entry, no changelog cut)

### Step 7: `main()` argument/validation paths (via main([...]))
- `--notes <version>`: prints notes, returns 0, mutates nothing
- invalid semver: returns 1
- missing marketplace.json under root: returns 1
- stable + prerelease version: returns 1
- beta version without `-` suffix: warns (stderr) but returns 0 and bumps
- happy path stable: returns 0 and applies all effects

### Step 8: verify
- `python3 scripts/test_release.py` green
- `python3 scripts/measure_coverage.py` → release.py well above 0% floor, gate green
- full suite still green (no cross-file regression)

## Folded Iron Council concerns (5/5 ENDORSE_WITH_CONCERNS)
- **Date determinism (skeptic/guardian):** assert the rolled heading via
  `datetime.date.today().isoformat()` computed in the test (or `## [<ver>] - ` +
  date regex), never a literal.
- **Trim Step 7 happy-path (pragmatist):** `cut_release` (Step 6) already pins all
  three effects; `main()` happy-path asserts `return 0` + one smoke check.
- **Fixture fidelity (architect):** marketplace seeds BOTH a channel entry (name NOT
  `^spec-loop-\d`) AND an existing archive entry (`^spec-loop-\d`) so `insert_at`
  lands between; a `seed_changelog(with_unreleased=)` knob covers the roll rewrite
  AND the `n==0` WARNING no-op; missing-marketplace case seeds a root WITHOUT the
  file.
- **Self-enforcing isolation (guardian):** tmp-root helper asserts root is under
  `tempfile.gettempdir()`; `--notes`/error cases re-read fixtures and assert content
  UNCHANGED (mutates nothing).
- **Repo idioms (historian):** use `contextlib.redirect_stdout/redirect_stderr(io.StringIO())`;
  always call `release.main([..., "--root", str(tmp)])` — pass `--root` even on
  validation/error paths.
- The sibling `RealRepoTest` regression guard is DELIBERATELY omitted (release.py
  mutates; constraint (c) forbids touching the real repo).
