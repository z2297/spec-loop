## Task 1 — CI hardening: read-only commands must not grant Edit (TDD)

The read-only contract on dashboard/dashboard-serve/peer-review is currently convention-only.
Add a stdlib check to `validate_marketplace.py` that, for every `commands/*.md`, decides
whether the command is marked read-only (frontmatter description OR prose contains a
read-only marker) and, if so, asserts its `allowed-tools` frontmatter does NOT include `Edit`.

Read-only marker detection (conservative, low false-positive):
- A command is read-only iff its FULL TEXT contains a read-only marker phrase. Marker list
  hoisted to a named module constant `READONLY_MARKERS` for auditability (Guardian, Historian).
  Folded down to discriminating phrases: `read-only` / `read only` / `never edits` (Pragmatist —
  `mutates no` / `changes no implementation` match zero files `read-only` doesn't already catch).
- The `Edit`-membership check parses `allowed-tools` ONLY from the frontmatter dict value
  (`fm["allowed-tools"]`, a JSON-ish one-line array preserved by `read_frontmatter`), NOT from
  full file text — this prevents a prose false-positive (peer-review.md prose literally says
  "`allowed-tools` excludes `Edit`"). (Architect.) Extract quoted tokens; compare each with
  exact equality `== "Edit"` (NOT substring) so a future `MultiEdit`/`EditNotebook` does not trip
  it. (Architect.)
- A best-effort docstring states the guard is keyword detection, not a sandbox. (Guardian.)
- Fail when a read-only command's `allowed-tools` contains `Edit`.
- Do NOT fail a command that has no read-only marker (quality-gate genuinely grants Edit with no
  marker; spec-loop has no marker and no Edit). Do NOT fail a read-only command that omits `Edit`.

Must not weaken any existing check. Stdlib only. Match existing `self.err(...)` style and the
existing frontmatter reading approach (but the minimal `read_frontmatter` parser only keeps
top-level keys and strips the value — `allowed-tools: [...]` value is preserved as a string,
which is what we parse). Read-only detection scans full file text (frontmatter + prose).
