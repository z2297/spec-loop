# Plan — slice s1: Generalize dashboard_server.py for Docker + multi-root

## Goal (scoped to THIS slice only)
Generalize the single dashboard server so it can run inside a container and serve
multiple data roots, WITHOUT weakening any existing guarantee. One server, extend
`build_server`/`main`/`scan_runs`. Two coupled concerns on the same file, done as
sequential TDD steps.

## Non-negotiable invariants (must-not-regress)
- Host-header allowlist stays CLOSED (never `*`, never container hostnames); it is
  derived from the ADVERTISED port, never widened by binding `0.0.0.0`.
- GET/HEAD only (405 otherwise). Path containment via `resolve_within`
  (`realpath`+`commonpath`) preserved for EVERY root. Bounded reads unchanged.
- Single-root behavior is the default and byte-for-byte back-compatible:
  `python3 scripts/dashboard_server.py --root .` works; `/api/runs` shape is
  unchanged when there is exactly one root (namespacing is transparent).

## Iron Council verdict (folded)
ENDORSE_WITH_CONCERNS (4 EWC + 1 non-SAFETY guardian OBJECT, folded — not a
council OBJECT). Folded concerns, now binding on execution:
- **Allowlist stays hardcoded loopback** — `_host_allowlist` keeps
  `127.0.0.1`/`localhost` host strings; `bind_host` is NEVER fed into it. Assert
  the allowlist set contains NO `0.0.0.0:*` entry, not just that a foreign host 421s.
- **Namespaced-id parse is untrusted input** — split on the FIRST `:` only; look up
  the owning `root_key` exactly against the per-root known-id set; confine the
  remainder via `resolve_within(owning_data_root, ...)` scoped to that ONE root.
  Never call `resolve_within` against a sibling root. Preserve the uniform
  no-path-oracle 404.
- **Param count** — keep `bind_host`/`advertise_port` keyword-only with defaults so
  `build_server`'s positional surface stays low (quality-gate params<=4); do NOT add
  a 5-positional signature. Keep `scan_runs(docs_root)` single-arg; add
  `scan_all_roots` alongside.
- **Deterministic root ordering** — root_key derived from sorted roots; collisions
  de-duplicated by a deterministic index suffix; pin stability with a test.
- **Single-root byte-for-byte** — bare `run_id`, NO `root` field, unchanged
  `{"runs":[...]}` envelope; the `root` field + namespacing appear only when >1 root.
  Guard with a regression test written BEFORE generalizing.

## Design decisions
### Part A — bind host / advertised-port decoupling
- `build_server(root_or_roots, assets_dir=None, port=DEFAULT_PORT, *, bind_host="127.0.0.1", advertise_port=None)`
  (bind_host/advertise_port keyword-only, keeping the positional surface at 3).
  `advertise_port=None` -> defaults to the effective bound port (so ephemeral
  port 0 still fixes the allowlist to the real port for tests).
- Allowlist derives ONLY from `advertise_port` via `_host_allowlist(advertise_port)`,
  which stays hardcoded to `127.0.0.1`/`localhost`. Binding `0.0.0.0` does not touch it.
- `main()` gains `--bind-host` (default `127.0.0.1`) and `--advertise-port`
  (default = `--port`). When `--port 0`, advertised port falls back to the real
  assigned port unless `--advertise-port` was given explicitly. (`--advertise-port`
  is named in the slice goal; retained despite the pragmatist's YAGNI note.)

### Part B — multi-root
- `scan_runs` gains a multi-root path WITHOUT breaking its single-arg signature:
  keep `scan_runs(docs_root)` working (single `docs/spec-loop` dir). Add
  `scan_all_roots(roots)` that aggregates, where `roots` is an ordered list of
  `(root_key, docs_root)`. Single root -> `root_key` is empty and run-ids are NOT
  namespaced (transparent / back-compat). Multi-root -> run-ids are namespaced as
  `<root_key>:<run_id>` deterministically (root_key derived from a stable,
  collision-free label per root; default = the root's basename, de-duplicated by
  index suffix if basenames collide).
- Response shape: `/api/runs` returns `{"runs": [...]}` unchanged for single root.
  For multi-root, each run dict carries an added `root` field (the root_key) and
  its `run_id` is the namespaced id; the collection stays a flat list so existing
  consumers that read `run_id`/`slices` keep working.
- Detail route `/api/runs/<id>`: `<id>` is the (possibly namespaced) id. Resolution
  enumerates known ids across ALL roots and confines within the OWNING root only —
  a run in root A can never resolve a dir in root B or escape either root.
- `build_server` accepts either a single root (str/Path, current callers) or a list
  of roots. It computes a list of `(root_key, data_root)` where each `data_root` is
  `realpath(root/docs/spec-loop)` — the per-root trust boundary, unchanged.

## TDD steps (Red -> Green -> Refactor each)
1. **Allowlist decoupled from bind host.** Test: `build_server(tmp, port=0,
   bind_host="0.0.0.0")` still 421s a foreign/`*` Host and 200s the advertised
   `127.0.0.1:<port>`/`localhost:<port>`. Then add `bind_host`/`advertise_port`
   params; allowlist from advertised port only.
2. **CLI flags.** Test `main`/arg parsing accepts `--bind-host` and
   `--advertise-port` and defaults correctly (advertise = port; port 0 -> real).
   Keep it a thin wire-through; assert via building a server, not by starting one.
3. **Single-root transparency (back-compat).** Test: one root -> `/api/runs` run
   dicts have the bare `run_id` (no `root:` prefix), and `scan_runs(docs)` still
   returns the same shape. Guard the existing behavior before generalizing.
4. **Multi-root aggregation + namespacing.** Test: two roots each with a run named
   `run-x` -> two distinct namespaced ids, no collision; each carries its `root`
   key; detail route resolves each to ITS root.
5. **Per-root containment across roots (adversarial).** Tests over a real socket:
   (i) a namespaced id pointing at root A cannot fetch a dir that only exists in
   root B; (ii) a crafted id like `rootA:../rootB/run-x` and a basename containing
   `:` or `/` return a uniform 404 with no path oracle (same body as a plain miss,
   per the existing `test_404_body_has_no_path_oracle`) and never read the sibling
   root; (iii) foreign/`*` Host still 421 with `bind_host="0.0.0.0"` AND the
   allowlist set contains no `0.0.0.0:*` entry; (iv) root_key ordering/de-dup is
   deterministic and stable.
6. **Refactor / simplify** to keep methods within quality-gate thresholds
   (cyclomatic <=10, cognitive <=15, method_lines <=50, params <=4, nesting <=3).
